# 📖 A股行情中心 - 使用指南

## 🚀 快速开始

### 1. 启动应用

```bash
streamlit run stock_app.py
```

### 2. 用户登录

- 在左侧边栏输入用户名
- 点击"登录/注册"按钮
- 新用户自动注册，初始资金：¥1,000,000

## 📑 功能导航

### 🔍 行情查询

**功能：**
- 查看A股市场指数（上证、深证、创业板）
- 搜索股票（支持中文名称或代码）
- 查看股票K线图和详细数据
- 查看公司简况

**使用方法：**
1. 在搜索框输入股票名称或代码（如：茅台、600519）
2. 从搜索结果中选择股票
3. 或使用"快速选择热门股票"下拉框
4. 查看实时价格、K线图、历史数据
5. 点击"查看公司简况"了解公司信息

### 💰 模拟交易

**功能：**
- 买入股票
- 卖出股票
- 自动获取当前价格
- 资金检查

**买入流程：**
1. 输入股票代码（如：600519）
2. 输入股票名称（如：贵州茅台）
3. 输入数量（最小100股）
4. 输入价格（可自动获取当前价）
5. 点击"🟢 买入"按钮

**卖出流程：**
1. 输入股票代码
2. 输入数量
3. 输入价格
4. 点击"🔴 卖出"按钮

**注意事项：**
- 买入时会检查资金是否足够
- 卖出时会检查持仓是否足够
- 交易单位：100股的整数倍

### 📊 我的持仓

**功能：**
- 查看所有持仓股票
- 实时盈亏计算
- 总资产统计

**显示信息：**
- 总资产 = 可用资金 + 持仓市值
- 持仓市值 = 所有股票当前价值总和
- 可用资金 = 可用于交易的现金

**持仓详情：**
- 股票代码和名称
- 持仓数量
- 成本价（买入均价）
- 当前价（实时价格）
- 市值（当前价 × 数量）
- 盈亏金额和盈亏率

### 📜 交易记录

**功能：**
- 查看所有交易历史
- 按时间倒序排列
- 显示买入/卖出详情

**记录信息：**
- 交易时间
- 股票代码和名称
- 交易类型（买入/卖出）
- 数量
- 价格
- 金额

## 💡 使用技巧

### 1. 快速交易

在"行情查询"页面查看股票后：
1. 记住股票代码和当前价格
2. 切换到"模拟交易"页面
3. 输入代码，价格会自动填充
4. 输入数量，点击买入

### 2. 持仓管理

定期查看"我的持仓"：
- 关注盈亏情况
- 及时止盈止损
- 调整持仓结构

### 3. 交易分析

通过"交易记录"：
- 回顾交易历史
- 分析交易策略
- 总结经验教训

## 📊 数据说明

### 价格数据
- 来源：新浪财经API
- 更新频率：实时（缓存1分钟）
- 数据准确性：与同花顺一致

### 交易规则
- 初始资金：¥1,000,000
- 最小交易单位：100股
- 交易费用：暂不收取（模拟）
- T+0交易：当天买入可当天卖出

### 数据存储
- 数据库：SQLite（trading.db）
- 数据持久化：重启不丢失
- 备份建议：定期备份trading.db文件

## ⚠️ 注意事项

1. **模拟交易**
   - 这是模拟交易系统，不涉及真实资金
   - 价格为实时行情，但交易为模拟

2. **数据安全**
   - 用户数据存储在本地数据库
   - 建议定期备份trading.db文件

3. **价格延迟**
   - 实时价格可能有几秒延迟
   - 建议以实际成交价为准

4. **交易限制**
   - 必须登录才能交易
   - 资金不足无法买入
   - 持仓不足无法卖出

## 🔧 常见问题

### Q: 忘记用户名怎么办？
A: 可以创建新用户，或查看trading.db数据库中的users表

### Q: 如何重置账户？
A: 删除trading.db文件，重新启动应用

### Q: 持仓盈亏如何计算？
A: 盈亏 = (当前价 - 成本价) × 数量

### Q: 可以多个用户同时使用吗？
A: 可以，每个用户独立账户，互不影响

### Q: 数据会丢失吗？
A: 不会，数据存储在SQLite数据库中，重启应用数据仍在

## 📞 技术支持

如有问题，请检查：
1. 是否已登录
2. 网络连接是否正常
3. 股票代码是否正确
4. 资金/持仓是否足够
5. trading.db文件是否存在
